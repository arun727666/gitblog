1. What is Git??

Git is a distributed version control system. It tracks your code changes over time and helps teams work on the same project safely.

  • Track file history
  • Create branches for features
  • Collaborate using pull requests

2. Basic Setup

Run these once on your machine:

git config --global user.name "Your Name"
git config --global user.email "you@example.com"
git config --global init.defaultBranch main

3. Basic Git Commands

Initialize: git init

Check status: git status

Add files: git add .

Commit: git commit -m "First commit"

Add remote: git remote add origin <repo-url>

Push: git push -u origin main

Pull: git pull origin main

View log: git log --oneline

4. CI/CD in Simple Words

CI/CD means your code gets automatically tested and deployed when you push changes.

  1. CI (Continuous Integration): Run lint, build, and tests on each push.
  2. CD (Continuous Delivery/Deployment): Deploy automatically after checks pass.

Common tools: GitHub Actions, GitLab CI, Jenkins, Azure DevOps.

5. Minimal GitHub Actions Example

name: CI
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Run checks
        run: echo "Run your tests here"
